聚连Tool · 自托管服务统一客户端 · (English: ConfluoTool)

连接指南

音乐（Navidrome / Subsonic）

兼容所有 Subsonic 协议的服务器：Navidrome、Funkwhale、Airsonic、Gonic 等。

  1. 在 Navidrome 网页端：创建一个用户（或用现有用户），记下密码。
  2. 在 app 内：音乐 tab → 添加音乐库 → 填服务器地址（如 https://music.example.com）、用户名、密码。
  3. app 验证通过后开始索引你的曲库。

常见坑：服务器地址必须能从手机网络访问；自签名证书需要在 iOS 设置里手动信任；Navidrome 路径前缀是 /rest/，app 会自动处理。

视频（Emby）

  1. 在 Emby：控制台 → 用户 → 新建用户（或用现有账户）。
  2. 在 app 内：视频 tab → 填服务器地址、用户名、密码。
  3. 服务器返回访问令牌后登录完成。

常见坑：需要 Emby Server 4.7+；反向代理必须转发 X-Emby-Authorization 头；强烈建议使用 HTTPS。

笔记（Memos）

  1. 在 Memos 网页端：设置 → 访问令牌 → 创建。
  2. 复制令牌。
  3. 在 app 内：笔记 tab → 粘贴服务器地址和令牌。

常见坑：app 通过 ConnectRPC 接口对接 Memos 0.18+；自托管旧版本请先升级。

财务（Firefly III）

  1. 在 Firefly：个人资料 → OAuth → 个人访问令牌 → 创建令牌。权限选：readonly（只读）或完整权限（读写）—— app 需要写权限来增改交易。
  2. 在 app 内：财务 tab → 填服务器地址，粘贴令牌。

常见坑：部分部署的 API 路径是 https://your-firefly.com/api/v1，有的就是 https://your-firefly.com——app 都接受。

常见问题

app 除了我的服务器，还会把数据发到别处吗？

不会。详见隐私政策。

有 Android 版吗？

暂时没有。iOS 是当前主平台。

能添加多个后端吗？（比如两台 Navidrome）

可以。音乐可以添加多个音乐库，指向不同服务器。侧边栏入口仍显示"音乐 / Navidrome"，因为这是产品族。

后台播放不工作 / CarPlay 连不上

确认 app 允许后台运行（iOS 设置 → 聚连Tool → 后台 App 刷新）。CarPlay 需要车机支持该协议，且 iPhone 首次连接时需解锁。

为什么让我订阅？

免费版覆盖核心功能。订阅解锁高级便利功能（离线下载、多库同步等）。具体差异见设置 → 订阅。不订阅也能一直用。

想提建议 / 反馈 bug

去 GitHub 提 issue（链接见下方）。带屏幕录制和你服务器版本的 bug 报告修得最快。
